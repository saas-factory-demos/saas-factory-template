# 服務條款範本（ToS Template）

> 對應 ADR-0100 / goal-11。每位透過 SaaS Factory 交付的客戶於簽收前必須閱讀並書面同意本條款。
> 條款編號：v2026-05-27

---

## 1. 雙方定義

- **服務提供方**（以下簡稱「**工廠**」）：______（公司／個人名稱）
- **客戶**：______（公司／個人名稱）
- **客戶站**：工廠依本案合約為客戶建置並部署之網站，含其 GitHub repository、Vercel project、Neon Postgres、Cloudflare R2 bucket、Sentry project 等構成基礎設施。

---

## 2. 交付範圍（B 買斷模式）

- 工廠完成客戶站建置、首次上線、13 個月免費保固維護
- 13 個月後客戶可選擇：
  - 續約年度維護服務（依下列條款持續享有技術支援）
  - 不續約（工廠仍依本條款保留必要的災難復原存取權，詳見第 4 條）

---

## 3. 客戶資產所有權

- 客戶站之原始碼版權、資料庫內容、媒體檔案、品牌設計，**自交付日起歸客戶所有**
- 客戶可隨時將以下平台資源完整移交（**ownership transfer**）：
  - GitHub repository（透過 GitHub 原生 transfer 流程，30 天緩衝期）
  - Vercel project（透過 Vercel team transfer，7 天緩衝期）
  - Neon Postgres project（即時 transfer，需客戶提供新 Neon org）
  - Cloudflare R2 bucket（透過 Cloudflare 原生流程）
- 客戶於完成所有 ownership transfer 後，工廠之 L3 平台層存取自動終止

---

## 4. 工廠技術支援存取條款（**重點條款，請務必詳閱**）

### 4.1 服務帳號（L1 應用層）

工廠於客戶站建置時，會在客戶站 Payload 後台建立一個**服務帳號**：

- **角色**：`factory-support`（系統權限與 admin 相當）
- **Email**：`support+<您的 subdomain>@<工廠網域>`（例：`support+demo-shop@saas-factory.app`）
- **用途**：用於工廠端執行下列維護動作：
  - 安全補丁批次套用（如 v1.4 → v1.5 升級）
  - 緊急故障排除（金流異常、結帳卡關、資料庫無法連線等 P0／P1 事件）
  - 客戶請求協助時直接登入後台處理

### 4.2 客戶可隨時停用此服務帳號

- 客戶可透過工廠端 `disable` 流程，將此帳號凍結；凍結後該帳號**無法登入**客戶站後台
- 客戶請寄信至維護窗口（合約附件 A）申請停用，工廠承諾 1 個工作日內處理
- 服務帳號可隨時恢復（`enable`）

### 4.3 客戶**不可**於後台 UI 直接刪除此服務帳號

- 為避免客戶端意外刪除導致無法接受工廠提供之災難復原服務，Payload 後台已禁止刪除 `factory-support` 角色帳號
- 客戶若有正當理由（合約終止 / 完全 ownership transfer）需永久移除，請走第 6 條退場流程

### 4.4 平台層存取（L3）

工廠於下列雲端平台對客戶站專案保留 admin 權限：

| 平台 | 工廠角色 | 退場機制 |
|---|---|---|
| GitHub repository | repository admin | 客戶請求 transfer 後 30 天移交 |
| Vercel project | team member（admin） | 客戶請求 transfer 後 7 天移交 |
| Neon Postgres project | project owner | 客戶請求 transfer 後即時移交 |
| Cloudflare R2 bucket | account admin | 客戶請求 transfer 後即時移交 |

- 此存取權僅於下列情境使用：
  - 災難復原（DB 損毀、deploy 失敗、SSL 過期等）
  - 安全事件處理（被入侵時隔離 + 修復）
  - 客戶授權之維護工作
- **客戶按下 4.2 之 disable 後，工廠仍保有 L3 平台存取**；客戶若要徹底斷捨，請走第 6 條退場流程

### 4.5 客戶資料存取原則

- 工廠承諾：
  - **不**讀取客戶會員個資（姓名、電話、地址、訂單明細）除非客戶明示請求協助
  - **不**修改客戶業務資料（商品、訂單、會員）除非客戶明示請求協助
  - **不**將任何存取行為用於工廠以外之商業目的
- 違反上述承諾構成重大違約，客戶得立即終止合約並請求賠償

### 4.6 稽核透明揭露

- 工廠端每次對客戶站執行維護動作，都會在客戶站資料庫 `factory-support-logs` 表寫入一筆紀錄
- 紀錄欄位：時間、動作類型、操作者 email、來源 IP（去敏化保留 /24）、操作摘要
- **客戶於後台 owner / admin 角色可隨時查閱**此紀錄表
- 工廠每月寄送月度健康度報告予客戶，內含「**本月維修存取 X 次**」一欄，無存取則顯示 0 次

## 5. 法律準據

- 本條款依**中華民國法律**解釋，包括但不限於《個人資料保護法》（PDPA）
- 工廠承諾遵守 PDPA 第 16 條（特定目的內利用）與第 20 條（國際傳輸限制）
- 若客戶於歐盟地區有營業活動或處理歐盟住民個資，需另簽署 GDPR 增補條款

---

## 6. 退場（合約終止）流程

客戶若決定徹底終止與工廠之關係：

1. **書面通知**：寄信至維護窗口，明確聲明「終止 SaaS Factory 服務」
2. **L1 服務帳號移除**：工廠收到通知後 3 個工作日內，透過 `disable` 將 `factory-support` 帳號凍結；客戶於 owner 帳號下手動刪除該帳號（已解除 access.delete 限制）
3. **L3 平台 ownership transfer**：客戶提供新 GitHub org / Vercel team / Neon org / Cloudflare account，工廠依本條第 4.4 表所列時程移交
4. **資料保留**：移交完成後 30 天內，工廠保留客戶站備份（給客戶反悔用）；30 天後永久刪除
5. **稽核紀錄保留**：客戶站 `factory-support-logs` 表內容隨客戶站本體保留，工廠端不留副本

---

## 7. 違約處理

- 任何一方違反本條款，他方得書面通知；違約方應於 7 個工作日內改正
- 若違反第 4.5 條（客戶資料存取原則）或第 6 條（退場流程），無補救期，他方得立即終止合約並請求損害賠償

---

## 8. 修訂

本條款若有修訂，工廠應提前 30 日書面通知客戶；客戶於通知期內未提出書面異議者視為同意。
